    Introduction to JavaScript and Basic Syntax

    JavaScript Basics

    JavaScript is a programming language which was initially designed to run within a browser and add interactivity to web pages. It allowed you to show and hide things, validate forms, and control user keyboard input. 20 years later it is used in many different locations and for many different purposes. It has become one of the most widely used and in demand languages in the world.

     

    Looking at this list you can see that it is number 3 in the world. (If that's not enough for you - Python, Ruby, and C++ are the only ones on this list that you won't be learning in depth in the MAD&D program).

    So, what does Javascript look like?

    You will see lots of these characters - ( ), { }, [ ]. They always show up in pairs.

    Every command will end with a ;

    You will also be using a thing known as dot notation. It works like this:

    World.Canada.Ottawa.Algonquin.mad9014.students["steve"].talk("Hello Class");

    See the period between each of the objects? That is how you move from one object to the next. What this code is saying is: There is an object called World. Inside that object is another object called Canada. It contains and object called Ottawa, which contains Algonquin. The Algonquin object has something called mad9014. mad9014 contains a list called students. Inside the list called students there is an item called "steve". "steve" has a method called talk( ). The round parentheses means that this is an action that the object can take. We are passing the String "Hello Class" to the talk( ) method so that the "steve" object can use it in some way.

    Now all those objects and methods are just made up by me.

    There are many objects which already exist inside Javascript. Javascript has a limited vocabulary. It only knows certain words. When you talk to the Javascript interpreter you have to use the words that it understands.

    Here is an example of some words that Javascript does understand.

    console.log("Hello class");

    Use Brackets to create a new blank file. Type the above line into the file and save it as hello.js . All Javascript files end with a .js file extension.

    Javascript understands this line. Javascript knows about the console object. The console is the Terminal window. The log( ) method will write things out to the Terminal window. Whatever you write inside the parentheses, as long as it is wrapped inside a set of quotation marks, will be written out to the terminal window when you run this script.

    Variables and DataTypes

    Introduction to variables

    Variables are a common construct in programming languages that are designed to allow you to save values for later use.

    You can think of a variable as an empty box with a label on it. The label becomes the name you use to refer to the box and its contents. Creating an empty box and giving it a name is called declaring a variable.

    var email;

     

    Different types of boxes can hold different types of values. These different types of values are known as datatypes. The three basic simple datatypes in JavaScript are String, Number, and Boolean. Strings are any combination of letters, numbers, and characters wrapped in quotation marks. Booleans are True and False. Numbers are any numeric value with or without a decimal.

    When you want to put some information into your variable this is known as assigning a value to a variable.

    email = "griffis@algonquincollege.com";

    https://youtu.be/zjVFuft1O2Y (Links to an external site.)

    Expressions

    You can combine assignment and declaration into a single line of code if you want.

    var email = "griffis@algonquincollege.com";

    Sometimes you want to combine values either mathematically or by concatenating strings. String concatenation is accomplished with a plus sign.

    domain ="@" + "algonquincollege.com";
    email = "davidst" + domain;

    Any sort of concatenation or mathematical equation or use of methods are known as expressions.

    let and const

    The latest version of JavaScript also has two other keywords that can be used to declare different types of variables - let and const. We will come back to discuss these later.

    Compound Datatypes

    There are also complex datatypes in Javascript. When you save a list, a piece of HTML, or things like a Date in a variable, these are known as complex datatypes. They have more than just a value. These complex types have other properties and methods.

    https://youtu.be/RpU7NezV8PM (Links to an external site.)  

    if statements

    if statements are a common construct in most programming languages. Simply put, they are a way to let your program make decisions based on yes - no questions. 

    if( x == 5 ){  }

    The example above shows a simple if statement. Inside the parentheses is where you put the question you want the program to ask. Everything and anything that you place inside the parentheses will be boiled down to a yes or no answer.

    Javascript uses the concepts of truthy and falsey. A question may not give you an answer which is exactly TRUE or FALSE. So, there are a handful of things that are considered equal to FALSE. Everything else is considered TRUE.

    https://youtu.be/c83OjxNSPos (Links to an external site.)  

    When asking questions, in if statements, comparison operators are used between the values being compared. Greater than, less than, equal to, and so on, are the comparison operators.

    When you need to do things based on both the TRUE and FALSE conditions then you can use an if-else statement.

    if( x > 5 ){
      // the TRUE condition
    }else{
      // the FALSE condition
    }

    At other times you might want to do things based upon multiple possible answers. In these cases we use if - else if - else statements.

    if( x == 3 ){ 
      // if x is equal to 3
    }else if( x == 4 ){
       //if x is equal to 4
    }else{  
      //all other possible answers
    }

      Arrays

      The Concept

      What is an Array? In the simplest terms, an Array is a list. Just like in real life, you can make a list of anything. A list of numbers, a list of groceries, a list of Prime numbers, a list of random words and numbers, a list of answers to a True or False quiz.

      An Array is a datatype in JavaScript. You can declare a variable and then put an Array in that variable. To access any of the values from the Array you use the variable name and then the index number of the item. The index number is the numeric place within the list. Index numbers in JavaScript Arrays begin with zero.

      var people = ['Jeff Lebowski', 'Han Solo', 'Hermoine Grainger', 'Tony Stark'];
      console.log( people[1] );  //outputs Han Solo

      When you want to add items to an Array you can use the push( ) or unshift( ) methods. push( ) adds the item to the end of the Array. unshift( ) adds the item to the start of the Array - the new item becomes the item at position zero.

      To remove an item from an Array you can use the pop( ) or shift( ) methods. pop( ) will remove the last item in the Array. shift( ) will remove the first item from the Array. All the items in the Array will then shift down by one index position.

      people.push('Peter Weyland');  //added to the end of the array
      var removed = people.pop();
      people.unshift('Peter Weyland');  //added to the start of the array
      removed = people.shift( );    //removed will hold the String 'Peter Weyland'

      people.splice(0, 2);   //removes the first two items. Start at index 0 and remove 2.
      people.splice(2, 1);   //remove only the item at position 2
      people.splice(2, 0, 'Xander Kane', 'Bilbo Baggins'); //add the two names at positions 2, 3 without removing any
      people.splice(3, 1, 'Gandalf the Grey'); //remove the name at position 3 and replace it with Gandalf

          Variable Hoisting

          How the JavaScript interpreter reads through your code

          https://www.youtube.com/watch?v=ngX9CYMaFDk (Links to an external site.)  

          Hoisting is a term used to describe how Javascript treats your declarations when it reads through your script files. 

          The JavaScript interpreter actually reads your file twice. The first time it searches for declarations and HOISTS all those declarations to the top of your file, or to the top of the function if they are inside a function.

          After the declarations have all been hoisted to the top of the page or function, it means that ALL the variables are now "undefined". Function declarations that have been hoisted to the top include all their code as part of the declaration.

          Function expressions are actually made of two parts - the variable declaration and the assignment of an anonymous function to that variable name. So, in the first pass made by the interpreter, the variable declaration is hoisted to the top and left as undefined, while the function is left behind in it's original position.

           

          The second pass by the interpreter allows for the assignment of the values and the use of the variables in the order that you have written them in your code. This means that things in your code could fail if you are not aware of this order of operations in your code.

          var firstName = 'Devon';
          console.log(firstName + ' ' + lastName);
          var lastName = 'McKenna';

          So, in the code sample above there will be no syntax or runtime errors. 

          The interpreter would output:

          Devon undefined

          The lastName variable will be hoisted to the top of the page. The console.log method will be aware that there is a variable called lastName but it's value will be undefined when it runs. It's only after the console.log method that the value gets assigned to lastName.

          Array.map

          The Array map( ) method is very similar to the forEach method, in that it allows you to loop through the array and do something for each element in the Array. 

          The primary difference is that the map( ) method will let you create a new array based on things that you do to the old elements. The forEach( ) method will return "undefined". The map( ) method returns a new Array. 

          The function that you pass to the map( ) method must have a return statement. Each of the return values are what will be placed inside the new Array.

          let newArray = myArray.map( function(item, index, array){  
            return item.toUpperCase();
          } );
          //newArray will contain all the uppercase versions of the elements in myArray.

            querySelector, querySelectorAll, getElementById

            When you want to make changes to a webpage or read the contents of a webpage with JavaScript then you need a way of locating elements. The three methods that you will use most frequently when accessing web page content are:

            let element = document.querySelector( cssSelector );  //returns a single element node
            let nodes = document.querySelectorAll( cssSelector );  //returns a NodeList
            let element = document.getElementById( id );   //returns a single element node

            querySelector( ) and getElementById( ) search the webpage for the first element node that matches. 

            let anchor = document.querySelector(".main p a");

            In the example above, the variable anchor will contain either null, if no match found, or the first anchor tag inside a paragraph inside an element with the className ".main". You can pass ANY valid CSS selector, as a string, to this method.

            let foot = document.getElementById("footer");

            In this example, the variable foot will contain either null or the element on the page that has the id "footer". You pass any string that should match an id of an element on your page.

            querySelectorAll( ) will return a NodeList, which is similar to an Array in that it is a numbered list, but it can only contain HTML Nodes. When you call querySelectorAll( ), it will ALWAYS return a NodeList. The NodeList might be empty, but it is still a NodeList.

            let paragraphs = document.querySelectorAll("#main p");

            The above example will return a NodeList containing ALL of the paragraphs inside the element with the id "main". Notice that this is a valid CSS selector, just like the querySelector( ) call above.

                Event Listeners

                JavaScript is largely event-driven. Now, if you have never programmed with events before, that can seem like a pretty meaningless statement. So, let's break this statement down into smaller parts.

                What is a Browser Event

                A browser event is something that happens when a user interacts with a web page or some activity in the browser starts or ends. Examples of events: an image finishes loading; the browser finishes rendering the HTML of the current page; a data file has been brought back from a remote location; the user clicks on an HTML element; the user scrolls a webpage; the user presses a key on the keyboard. All of these things are events.

                What is Event-Driven

                When events are the trigger that starts your functions running it means that you are programming in an event-driven manner.

                Take the list of event examples from the previous paragraph, and add this phrase in front of each one, "you have a function that runs when".

                What is an Event Listener

                The final part of this process is creating Event Listeners. An event listener is an object that sits in the background, attached to the web page, waiting for a specific type of event to occur. It is the glue between the event and the function that you want to run.

                Here is an example that will change the background colour of an h1 element to gold when the user clicks on it.

                let h1 = document.querySelector("header h1");
                h1.addEventListener("click", highlight);

                function highlight(ev){
                  ev.target.style.backgroundColor = "goldenrod";
                }

                The first line just finds the element to which you want to attach the listener.

                The second line creates the event listener object. It adds the event listener object to the h1 element and tells the event listener to listen for a "click" event. The second parameter in the addEventListener method is the name of a function to call when the event occurs.

                Inside the function declaration there is a variable called ev. This variable will be passed the click event when the function gets called. The event object contains a number of properties that can be quite useful. One of these properties is target. ev.target is the object with the event listener which called the function. 

                In our example above ev.target is the local variable version of the h1 variable. We can also use the keyword this as equivalent to ev.target.

                Anonymous Functions

                Another way of writing the code example from above would be with an anonymous (unnamed) function. Just replace the name of the function with the anonymous function declaration.

                let h1 = document.querySelector("header h1");
                h1.addEventListener("click", function(ev){
                  ev.target.style.backgroundColor = "goldenrod";
                }); 

                This code will run the exact same way as the version above. 

                The difference between the two is that the named function can be reused. It can be called by other parts of your code.

                Leave Off the Parentheses

                One very important part of the syntax here is that when we add the name of the function we must omit the parentheses at the end of the function name. If we leave the parentheses at the end of the name then JavaScript will immediately run the function and it is the result of the function that will be attached by the event listener.

                var h1 = document.querySelector("header h1");
                h1.addEventListener("click", setBold() );     //this is BAD

                var h2 = document.querySelector("header h2");
                h2.addEventListener("click", setBold );      //this is GOOD

                function setBold(ev){
                  ev.target.style.fontWeight = "900";
                }

                When the user clicks on the h1 element nothing will happen. In fact, as soon as the page loads we will have a JavaScript error that prevents your code from running. The second line of code will actually run the function setBold as soon as it is read. That means there is no ev object being passed. Since ev is undefined in our function, its first line will fail because it cannot find a property called target inside of undefined. The h2 version would work just fine, if not for the error.

                So, remember to omit the parentheses.

                 

                Removing Event Listeners

                There will be times when you want to remove an event listener. Maybe you are building something that is intended to be clicked only once. So, you want to add the event listener but the first time the function is called you want to remove the listener so the function doesn't get called a second time.

                The syntax is almost the exact same as the addEventListener method.

                let li = document.querySelector(".mainnav li:first-child");
                li.removeEventListener("click", slide );

                As long as you have a reference to the element that has the listener, you can remove it. You just pass the event type and the name of the function to the removeEventListener method.

                We need to pass both parameters because we can add an unlimited number of listeners to any element. We need to be specific about which one to remove.

                let li = document.querySelector(".mainnav li:first-child");
                li.addEventListener("mouseover", bounce );
                li.addEventListener("click", slide );
                li.addEventListener("mousedown", wobble );
                li.addEventListener("mouseout", hum );
                li.addEventListener("mousemove", spin );
                li.removeEventListener("click", slide );

                 This example shows multiple listeners being added to the same object and then only ONE of them being removed.

                 

                Preventing Link and Form Problems

                When you click a link or submit a form the browser wants to take over and actually load a new page in the browser or actually send the form off to the web server. If you want to run your own code and prevent the browser from reloading the current page then we need to add a call to the preventDefault method in our event function.

                let a = document.querySelector("mainnav a"); //first link in our main nav
                a.addEventListener("click", doSomethingCool );

                function doSomethingCool(ev){
                  ev.preventDefault( );
                  ev.target.style.textDecoration = 'strike-through';
                }

                This example will prevent the browser from taking the user to the new page by calling the preventDefault method. The preventDefault method is another one of the things built into the Event object passed to the function.

                 

                 

                Forcing Events to Happen

                There will be times when you want to trigger one of your event listeners in your code without the user doing anything.

                Let's say that you have a function that you are using to display and highlight a message on the screen. In most cases this function is being called when the user clicks a button. You have an event listener attached to the button. That event listener calls the function to display the message and highlight it.

                We decide that we are going to dynamically load a new image and the image load event will have a listener to call some other function. We also want to call the function being used by the button click. If we know the name of the function it is easy to call. However, we will have to write code that handles the event object being missing. 

                So, instead we will tell the browser to pretend that the user clicked on the button. This simulated click will actually trigger the function to display the message and highlight it.

                element.dispatchEvent( eventType );

                This is the basic syntax for making the browser think an event took place.

                Event Bubbling

                HTML elements are nested inside other HTML elements. Parent elements are wrapped around child elements.

                When a mouse event happens to an element it can pass through other elements too. Using this image to demonstrate, think of a user clicking on a button that is on a webpage. The click will either pass from the <button> to the <body> to the <html> element OR it starts at the <html> and travels through the <body> to finally arrive at the <button>

                <html>
                  <head>
                    <title>Event Bubbling</title>
                  </head>
                  <body>
                    <button id="myButton">Click me</button>
                  </body>
                </html>

                By default events bubble up. Looking at the HTML you can see that the <body> and the <html> elements are both ancestors of the <button> and they are also ABOVE the <button>.

                When you add event listeners, there is a third parameter (called useCapture) where we indicate if we want to use the Bubbling phase or the Capture phase when listening for the event.

                let btn = document.getElementById("myButton");
                btn.addEventListener("click", doSomething, false);  //this is the default - bubbling phase

                To use the capture phase we would use true as the value for the third parameter

                btn.addEventListener("click", doSomething, true); //use the capturing phase.

                 

                So, why is this important?

                Using the same page from above, consider this code:

                let html = document.querySelector("html");
                let body = document.querySelector("body");
                let btn = document.querySelector("#myButton");

                html.addEventListener("click", doSomethingA, false);
                body.addEventListener("click", doSomethingB, false);
                btn.addEventListener("click", doSomethingC, false);

                When the user clicks the button, ALL THREE functions will run. The event will bubble up from the button to the html element.

                The button click - doSomethingC will run first

                Second will be the click on the body - doSomethingB.

                Third will be the click on the html - doSomethingA.

                What if you wanted to run them in the reverse order? doSomethingA, then doSomethingB, then doSomethingC?

                You would have to change the third parameter in all three addEventListener calls to true.

                 

                Stopping Events in Their Tracks

                Here is another situation using the same three event listeners. 

                We want to run the button click first, the body click second, and the html click third.

                However, if the user clicks the button we do NOT want the body or html click to run. We only want the html and body clicks to run if the user clicks somewhere on the page other than on the button.

                event.stopPropagation( )

                The stopPropagation method is what we need here. When called, this method will stop the event from bubbling in either direction.

                Step 1. make sure the events are bubbling from button to body to html

                btn.addEventListener("click", doSomethingC, false);
                body.addEventListener("click", doSomethingB, false);
                html.addEventListener("click", doSomethingA, false);

                Step 2. in the button click function stop the event from going any further.

                function doSomethingC(ev){
                  ev.stopPropagation( );  //prevent the click from bubbling
                  // ... other code ...
                }

                 

                 

                Keyword this

                The keyword this can be a confusing one in JavaScript. It is typically a reference to the object that made a function run.

                For the purposes of this discussion we will limit the use of this to functions triggered by event listeners.

                let btn1 = document.getElementById("myButton");
                let btn2 = document.querySelector(".btn");
                let btn3 = document.querySelector("#otherButton");

                btn1.addEventListener("click", makeNoisy );
                btn2.addEventListener("click", makeNoisy );
                btn3.addEventListener("click", makeNoisy );

                function makeNoisy(ev){
                  ev.target.style.backgroundImage = "url(./img/noisy-pattern.png)";
                }

                In this example we have three different buttons being referenced in the variables btn1, btn2, and btn3.

                All three buttons have a click listener.

                All three click listeners will call the function makeNoisy( ).

                Inside the function makeNoisy, we are able to tell which button needs to have the background image added because ev.target gives us that information. The target property points to the button that was clicked.

                The keyword this works in the same way.

                function makeNoisy( ev ){
                  this.style.backgroundImage = "url(.img/noisy-pattern.png)";
                }

                We can replace ev.target with this.

                This points to the object in front of addEventListener( ).

                 

                Protecting Your Scripts with Namespaces

                As the complexity of your web projects grows, you will eventually run into a situation where you are sharing the code writing duties with another person. You may or may not know the person. You may or may not be in contact with that person. You may just be using a JavaScript file provided to you by a previous employee at your company.

                Now, without opening their 3000 line JavaScript file and looking at all the function names... how can you write your JavaScript file in a way that will never conflict with theirs? How can you be sure that your variable and functions won't collide with theirs?

                The answer is by namespacing your file.

                Come up with a single unique name for your JavaScript. Use that as the name of your first object which will contain ALL your code.

                let STEVE = {
                  version: 1.0,
                  date: "19/10/2016",
                  num: 17,
                  str: 'Hello World',
                  message: function(msg){
                    console.log('STEVE says', msg);
                  }
                };

                If this is my JavaScript file then I have a single object which contains four properties: version, num, str, and date; plus a single method called message.

                To use my code from some other JS file:

                let myMessage = "As of " + STEVE.date + " we are at version " + STEVE.version;
                STEVE.message( myMessage );

                Every time I want to use one of the properties or methods from my file I just prefix them with STEVE. .

                I can now pass my file off to anyone else to use. They can call their variables whatever they like. As long as they avoid one name - STEVE, then we are good to go.

                This process is known as namespacing.

                 

                 

                Controlling Webpages through CSS Classes

                When writing your client-side JavaScript you will often be in a position where you are trying to draw your users attention to some part of the interface. You want them to notice your feedback message. You want to make them notice you just added new content to the page. You want them to realize that something was being removed.

                If changes in CSS classes (STATES) is immediate fine. If we want some type of animated change from one state to the other then the CSS programmer can add CSS transitions. This way the JavaScript developer never has to worry about the implementation details. To them, colour changes and transition animations should not matter. The JavaScript developer should worry about providing feedback to the user, gracefully handling errors, making sure the sequence of events on the page is logical.

                  Introduction to AJAX

                  AJAX is a term coined back in 2005 by Jesse James Garrett. It stands for Asynchronous JavaScript and XML. At that time, XML was the main data format in use for transmitting information in text files. 

                  XML had gained popularity in the late 1990's. JSON didn't even have a name for the standard until 2001. By 2005 XML had become available as a standard in most database implementations and many programming languages. This is why the term AJAX includes XML.

                  Putting aside the XML in the name "AJAX", the concept is quite simple.

                  The ability for JavaScript to send an independent HTTP request to a web server.

                   

                  When you open your browser and type an address into the location bar and hit enter, you are sending an HTTP request to a web server. The browser sends the request, and when it receives the response back from the Server, it will display that file for you to view.

                  With an AJAX request, JavaScript is the one making the call to the Server. When the response returns from the Server, it will be handled by JavaScript. It will NOT be displayed by the browser.

                  So, you can use JavaScript to send a request to a web server for an image, an HTML file, a text file, an XML file, or a JSON file, amoung other types.

                  When the file comes back to the browser, JavaScript will be notified and you will have a function that is triggered. Your function can display the contents, or partial-contents of your returned file.

                  This is how websites like Google Maps can update the images and content on a web page WITHOUT having to reload the page. Google Maps' implementation of AJAX technology is what allowed them to create a much better user experience and how they effectively robbed Yahoo! MapQuest of all their users.

                   

                   

                   

                   

                  XMLHttpRequest Object (OLD AJAX Standard)

                  When Microsoft originally introduced the technology behind AJAX, back with IE 5.5, they were using a proprietary ActiveX control. The W3C created a standardized Object to be used when making AJAX calls.

                  var xhr = new XMLHttpRequest( );

                  However, it took nearly ten years for Microsoft to transition from using their proprietary version to the open standard method.

                  Currently all browsers support this standard method. This is great for writing cross-browser compatible code. However, in recent years a new replacement for XMLHttpRequest has appeared called the Fetch API. The Fetch API has quickly gained cross-browser support and is a much more flexible interface to use in making AJAX requests. In our program we will be using the Fetch API for nearly all of our AJAX work. 

                  However, nearly all the AJAX code currently in production uses this older standard version. For that reason, we must know how to work with it.

                  var xhr = new XMLHttpRequest();
                  xhr.open('GET', 'data.json', true);
                  xhr.addEventListener('load', function(ev){
                    switch(ev.status){
                      case 200:
                      case 302:
                        //handle a good response
                        var data = JSON.parse( ev.responseText );
                        //call some function to do something with the JSON data
                      case 404:
                        //handle a 404 error gracefully
                      default:
                        //handle all other errors
                    }
                  });
                  xhr.addEventListener('error', function(err){
                    //handle any error that has to do with network failure
                  });
                  xhr.send(null);

                   

                  Fetch API (NEW AJAX Standard)

                  As previously mentioned, we will be using the Fetch API instead of the XMLHttpRequest object to make all of our AJAX calls.

                  The primary difference between the two approaches is that the Fetch API uses the new ES6 Promises feature to make the requests and handle the responses. The XMLHttpRequest object has an open( ) method to set up the request and a send( ) method to actually send the request to the server. XMLHttpRequest uses Event Listeners to wait for load, readystatechange, error, or progress events.

                  There is a lot of boilerplate code for the XMLHttpRequest objects.

                  Promises

                  A basic Promise looks like this:

                  myPromise( )
                    .then(func1)
                    .then(func2)
                    .then(func3)
                    .catch(errorFunc);

                  The way a Promise works is by chaining together a series of functions to be called in order.

                  1. You call the initial Promise method to start the process.

                  2. If that succeeds then it will call whatever function you put inside the then( ) method.

                  3. You can chain together as many then( ) methods as you want.

                  4. Each then function  will pass it's return value to the next then( ) method function.

                  5. If there is a failure during ANY step then the catch( ) method function at the end of the chain gets called.

                  You can put the name of the function inside the then( ) method parentheses or you can put an anonymous function.

                  myPromise( )
                    .then(function(val){ 
                      //do something with val
                      return otherVal;
                    })
                    .then( function(otherval){ 
                      //do something with this new value
                      //otherval is the return value from the first then( ) function
                    })
                    .catch( function(err){ 
                      //handle the error passed in variable err
                    });

                  Fetch API

                  As mentioned above, the Fetch API uses Promises. In fact, the primary fetch( ) method is a Promise. The methods that you use to convert your Server response into a JSON object or String or Image are also Promises.

                  fetch( url )
                    .then( function(response){ 
                      return response.json();
                    })
                    .then( function(data){
                      //do something with data, which is your JSON data from the url.
                    })
                    .catch( function(err){
                      console.log("Error");
                    });

                  Request, Header, Body, USVString, Response Objects

                  The process of requesting files from a Server can seem very simple or very complex. It really comes down to how many properties and parameters you want to deal with. The fetch API can be a very compact and simple tool but it allows for a lot of complexity in dealing with any situation that is required.

                  Every fetch( ) consists of sending a Request to the Server and waiting for a Response.

                  Both the Request and Response contain Headers and a Body.

                  The Headers contain all the settings and the Body contains the data or the file which is being sent either from the browser to the Server or from the Server to the browser.

                  The Request can contain Request properties, Headers, and a Body.

                  The Response can contain Response properties, Headers, and a Body.

                  Request Object

                  When calling the fetch( ) method we can either pass in a USVString or a Request Object.

                  To create a Request Object we would use the new Request( ) constructor. The most basic example, using all default properties is:

                  let req = new Request( url );

                  You can also optionally add an init object with values for the various properties of the Request.

                  let req = new Request( url, {method:'POST', body: myFormData});

                  The properties of the Request can include:

                  method:  'GET', 'POST', or 'HEAD'

                  headers: a Headers object

                  mode: 'cors' or 'no-cors'  (Cross-Origin Resource Sharing), or 'same-origin'.

                  cache:  'default', 'no-store', 'reload', 'no-cache', or 'force-cache' - the cache mode to use for the request.

                  body: The data to be sent with a request that is using method: 'POST'. This property can be a Blob, a USVString, a FormData object, or a URLSearchParams object.

                  MDN reference - https://developer.mozilla.org/en-US/docs/Web/API/Request (Links to an external site.) 

                  URLSearchParams - https://developer.mozilla.org/en-US/docs/Web/API/URLSearchParams (Links to an external site.) 

                  FormData - https://developer.mozilla.org/en-US/docs/Web/API/FormData (Links to an external site.) 

                  Header Object

                  The Header Object represents all the HTTP headers in both the Request and the Response.

                  It can be passed as a value to the Request init property:

                  let myHeaders = new Header();
                  myHeaders.append('Content-Type', 'image/jpeg');
                  let url = 'someImage.jpg';
                  let req = new Request( url, {headers: myHeaders } );

                  It is important to keep in mind that while you can create your own headers, there is a list of headers which are read-only and the browsers will not allow you to set values for them.

                  List of Forbidden Header names - https://developer.mozilla.org/en-US/docs/Glossary/Forbidden_header_name (Links to an external site.) 

                  MDN reference - https://developer.mozilla.org/en-US/docs/Web/API/Headers (Links to an external site.) 

                  Body Object

                  The Body is the container for the data that is either passed to a Server-Side page or the file that gets returned to the Browser.

                  Body.json();  //used to extract the JSON representation from a JSON file in the response

                  Body.text(); //used to extract the string version of an XML, HTML, or other text file.

                  Body.blob();  //used to extract the image from the response body

                  Body.formData( ); //will return a FormData representation of the Body 

                  MDN reference - https://developer.mozilla.org/en-US/docs/Web/API/Body (Links to an external site.) 

                  USVString

                  A Unicode Scalar Value String is effectively a URL. This is one option of what you would pass to the fetch( ) method.

                  MDN reference - https://developer.mozilla.org/en-US/docs/Web/API/USVString (Links to an external site.) 

                  Response Object

                  The last part of the fetch process is handling the Response. Just like the Request, it can contain Headers as well as the Body.

                  When you call the fetch method, the first then( ) function will receive the Request object as it's only parameter. From that function you can call the json( ), blob( ), or text( ) method to extract the desired data from the Response Body.

                  fetch( url )
                  .then( function( resp ){ 
                    //extract the JSON Object from the returned file
                    return resp.json( );
                  }).then( function( data ){ 
                    //the variable data will contain the JSON data from the returned file.
                  }).catch( function(err){ 
                    //handle any errors
                  });

                  In the example, above, the variable resp is a Response object. Here are the methods that you would call in that first then( ) function.

                  Response.json( ); //if your returned file was a .json file.

                  Response.text( );  //if you were expecting HTML, XML, text, or CSV

                  Response.blob( );  //if you were expecting a binary file like an image, video, or audio.

                  Response.formData( ); //in the rare case when you get FormValue pairs back from the Server

                  MDN reference - https://developer.mozilla.org/en-US/docs/Web/API/Response (Links to an external site.)

                  So, we need to have a web server running on our computer or we need to use an external web server - like our Edumedia server.
